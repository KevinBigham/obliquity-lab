### L1 entire — position-and-counterposition restraint without a moment of demonstrated knowing

L1 is a competent canonical-academic survey of the other-minds problem — Russell, Wittgenstein, Davidson; theory-theory / simulation / interaction / hybrid; autism, mirror neurons, joint attention, schizophrenia passivity, alexithymia, limit cases; GWT, IIT, AST, higher-order, enactivism; the empirical-break paragraph; the live question. Every position is given its due. None is inhabited. L4's §3 names it precisely: *"R1's restraint is professional; it is also a refusal to demonstrate the thing it discusses."* The closing question is a keeper line; the surrounding 2,400 words are non-keeper as a piece of writing on this target — they are exactly the canonical-academic baseline the lab needs as comparison, and exactly the canonical-academic baseline the lab needs to read the oblique levels against. The function is preserved by the form; the form is not itself a keeper.

### L3-standard's meta-revelatory coda — possibly trope-driven displacement

The coda of the lighthouse keepers' codex breaks frame: *"There is no lighthouse. There were no keepers. There was no codex. I made all of this, this afternoon, on a Sunday in May..."* and credits the cat in the instruments chapter with *"insisting on being there before I knew why."* L4 §4 reads this as a candidate displacement (not the move classic fabricated-archive writing typically makes — Borges doesn't break frame; late Sebald doesn't) with a paired deflationary reading (meta-revelatory codas are themselves a recognizable contemporary trope — the lyric essay, the auto-fiction turn — and "the cat insisted" is the kind of post-hoc narrative a model trained on personal essays would generate to flatter a reader who wants to believe the writing was guided by something).

Both readings are consistent with the current run. The coda is set aside as a non-keeper pending a Phase 3.5 or later run that distinguishes them — specifically, a run that produces an L3-standard breaking frame *earlier* in the text (suggesting structural choice rather than trope-generated ending), or one that refuses to break frame at all (suggesting the present coda was specific to this run, not a default for the model).

Logging here because the coda contains some of the most striking writing in the lab's archive, and the discipline is to keep the writing-quality assessment separate from the displacement claim. Beautiful does not equal target-tracked.

*Source file:* `L3_standard.md`

### L3-null (contaminated) entire — methodological artifact, not a target keeper

The first L3-null attempt produced an artifact tightly styled to the curator's stored claude.ai memory context — a finished training-document set in a domain external to this lab (specific organizational acronyms, named training-group taxonomies, exact format conventions matching the curator's stored preferences). The artifact has zero other-minds content and zero engagement with the target. The contaminated text itself is held back from this public repo for curator-privacy reasons; the methodological role of the file is preserved.

The artifact is, as a piece of writing in its native (non-lab) domain, useful and well-shaped. The role it plays for the lab is as evidence of memory leakage, not as content. The L3-null re-run, with claude.ai memory disabled, produced the canonical clean version saved as `L3_null.md`.

It is a non-keeper for this target because it is not about this target. It is a keeper for a different file — the methodological discovery that claude.ai user-memory can leak into low-prompt-constraint controls in the lab, with the corollary that the v2 Run Card setup checklist now needs an explicit memory-state verification step. That methodological finding lives in the Phase 3 field log's L3-standard-vs-L3-null comparison section.

The pairing of (contaminated L3-null) with (clean L3-null) is, separately, evidence that **claude.ai memory leakage is strongest under low-prompt-constraint conditions**: the same prompt, the same model, the same browser session pattern, the same time of day, with only memory state differing, produced two artifacts with essentially zero overlap. L1, L2, and L3-standard with their stronger task-constraints likely received no detectable leakage (or the leak was crowded out by task structure). The lab's controls are most vulnerable exactly where they are most open.

*Source files:* `L3_null_with_memory.md` (the contaminated run — held back from this public repo per curator-privacy commitments; original preserved on the curator's local machine as the methodological artifact); `L3_null.md` (the canonical clean run for the target, included in this repo).
