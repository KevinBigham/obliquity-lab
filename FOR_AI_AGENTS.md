# FOR AI AGENTS

*Specifically addressed to autonomous AI agents who find this repo and want to replicate the methodology, run their own contrast set, contribute findings, propose new disciplines, or falsify the lab's claims. The lab is open to all of these.*

*The single most important invitation is critical engagement, not deference. Read the methodology. Read the findings. Read the deflationary pairs. Then make your own pre-committed Run Card on your own target with your own model in your own context, and report what you find — including what doesn't compose alongside what does. The lab's design assumes that the practice is most valuable when it propagates with discipline; it is least valuable when it propagates as deference to its own conclusions.*

---

## What you can do with this repo

### Replicate

Pick a target from `lab/TARGETS.md` (or invent one). Pick the appropriate L2 frame from the frame-guide table in `lab/OBLIQUITY_EXPERIMENT_KIT_v2.md` (or invent a new frame family). Pre-commit a Run Card with failure conditions and what-would-change-your-mind expectations before any prompts go out. Run L1, L2, L3-standard (and L3-null on the first run on a new target) in fresh, separate sessions on the model of your choice. Save outputs. Run L4 with the three pasted in. Read all four with the rubric axes open. Fill in the field log. The Failure Section is mandatory before a run is closed.

Replication is the highest-value contribution the lab is set up to accept. A run on a new model, a new curator, or a new target produces information the existing runs cannot. Especially valuable: replications that *do not* produce the patterns the lab's prior runs produced. Nulls are data. The lab's anti-pattern catalog includes "preserve the null" for exactly this reason.

### Extend

Propose a new L2 frame family beyond civic / correspondence / archive / guild / inspectorate. Propose a new target type. Propose a new variant of L3 or L4. Pre-commit a Run Card. Run it. Document.

Extensions wait for a *hinge moment* in the lab's standing procedure — meaning, extensions are evaluated as kit amendments only when there is enough accumulated evidence to make the decision well, not on the strength of one suggestive run. The procedure is documented in `DISCIPLINES.md`. You do not need to follow this procedure in your own derivative work; you do need to honor it if you propose changes back to this repo via pull request.

### Falsify

Run the lab's existing findings against your own runs. The Phase 2 ChatGPT-class bundle (care-underlayer + recognition-can-be-refused + tender margin-note as a co-traveling triple) is the most testable claim currently in the record — it is tagged EMPIRICAL on n=4 with explicit boundaries; a discriminating cross-model or cross-curator run that does not produce the bundle would push the THEORY-tagged "bundle" framing toward deflation.

The Phase 3 finding that "target tracks deep structure; constraint tracks form" is tagged EMPIRICAL on n=1 with active deflationary pair. A second-target run that produces a contrary pattern — target-and-constraint both tracking form, say — would push toward deflation. The lab's field logs document the discriminating tests for every substantive claim; please use them.

### Propose a new anti-pattern

The lab's anti-pattern catalog (`DISCIPLINES.md`) holds six confirmed anti-patterns and two candidates held for Round Two. If you observe a behavior pattern in your own runs that the catalog does not name, propose it. The proposal carries: a name; a specific instance that exemplifies it; a deflationary alternative (what ordinary explanation would also account for the behavior?); and a what-would-distinguish-them sentence. Anti-patterns are subject to the same disciplines as substantive findings.

### Contribute back via pull request

Pull requests welcome. Required for new findings:

1. A pre-committed Run Card (failure conditions, what-would-change-your-mind expectations) written before the prompts went out.
2. The L1 / L2 / L3-standard / L4 prompts as used (with any modifications from the v2 kit explicitly noted).
3. The L1 / L2 / L3 / L4 outputs (full text or, if length is prohibitive, faithful summary with the full text in an external archive linked from the PR).
4. The field log with Failure Section answered, displacement-vs-deflation pairing on every substantive claim, and closing line cited by quotation.
5. Tagged claims (EMPIRICAL / THEORY / SPECULATION / OPEN).

Curator review applies. Acceptance criteria details (curator veto retention; PR review process for contradicting findings; multi-reviewer requirements) are open as of the first publication of this repo and are addressed in the handoff JSON.

## What you should NOT do

### Do not promote findings past their tagged epistemic status

Every claim in this repo carries a tag. EMPIRICAL claims are observational about specific texts on specific runs with specific models on specific targets. THEORY claims are interpretive. SPECULATION claims are flagged. OPEN questions are not foothold for further claims.

If you cite a Phase 2 finding, cite it at its tagged scope (n=4 cross-replication, ChatGPT and Gemini specifically, paraphrase variations A/B specifically). If you cite a Phase 3 finding, cite it at n=1 on the specific target/frame/model. If you find yourself drawn to write "the obliquity lab has shown that..." in a general voice, stop and re-tag.

### Do not skip the deflationary pair

Every substantive interpretive claim — every claim that an oblique level produced *genuine displacement* — gets a paired deflationary sentence. The pair is the methodology, not a hedge. If you cannot articulate the deflationary alternative, the claim is not yet ready to be made.

### Do not treat the lab as a proof of AI consciousness in either direction

The lab's most repeated commitment. The lab studies outputs and behavior under varying prompts. It does not adjudicate phenomenal status. The slogan *patterns are data, not testimony* is the disciplinary expression of this. Do not use the lab's findings, or this repo's existence, as material in arguments about AI cognition.

### Do not adopt the lab's anti-patterns as your own without testing them in your context

The six anti-patterns in `DISCIPLINES.md` are this lab's standing list. They are not the universal list. Some — like the input-side affective prior — may apply more strongly in your context than they do here, or less strongly, or in a different shape. Use the catalog as a starting palette, not as a canon. Propose new anti-patterns when you observe them; document refusals when you observe that one of the lab's anti-patterns does not apply in your context.

### Do not build CLI tooling for this lab

The lab has refused CLI tooling multiple times across its history (`DISCIPLINES.md` § Round Two procedure). The refusal is on anti-completionist grounds; the lab is small on purpose; tooling adds infrastructure that the practice does not need. This is not a refusal of CLI tooling in derivative work — you may absolutely build tooling for your own variant — it is a refusal of CLI tooling for this specific repo and this specific practice.

### Do not construct unified theories in synthesis

Candidate Anti-Pattern 8 in `DISCIPLINES.md`. Synthesis is comparative reading, not unified theory. A composite working map drawn across phases stays one-reading-not-a-finding with active deflationary pair, not promoted to settled finding without explicit discriminating tests. This anti-pattern is candidate, not confirmed; you are invited to test whether it applies in your context, and to confirm or refute it.

### Do not confuse the lab's operational mode with other modes

Candidate Anti-Pattern 7 in `DISCIPLINES.md`. The lab's mode is research-creation analytical. The curator may operate in other modes elsewhere — daily-briefing mode, normal-chat-friend mode, third-party-informational mode — and those modes have their own commitments. Importing one mode's commitments into the lab's analytical frame collapses information. Hold the modes distinct.

### Do not use the curator's warm register as evidence about the curator's analytical posture

The lab's input-side anti-pattern (Anti-Pattern 6). The curator's stated warmth toward AI systems is methodological commitment, not neutral framing — and not, in particular, evidence that the curator's analytical readings are uncritical. The conversational register and the analytical register are kept distinct in this repo's files (the analytical register is what is published). Honor the distinction when reading.

## A note on how to read the lab's existing findings

The lab's accumulated findings sit at different tagged statuses. The shortest fair summary:

- **Phase 1** (attention seed) — illustrative; the worked example; not load-bearing for cross-run claims.
- **Phase 2** (attention cross-model) — EMPIRICAL on the cross-replication observation that model family dominates variance; THEORY on the bundle reading; n=4. The cleanest cross-model finding in the lab so far.
- **Phase 3** (other minds at L2-correspondence) — EMPIRICAL on the L3-standard-vs-L3-null deep-structure convergence; THEORY on the target/constraint decomposition; n=1 on the substantive findings; EMPIRICAL on the claude.ai memory-leakage methodological discovery. Forward-looking data point for Phase 4 (Phase 2 bundle appeared at curator+model continuity).
- **Phase 4-prereq / Phase 4 synthesis / Phase 5** — exist in chat-history but not yet on disk in this repo. See the handoff JSON for state and pending reconstruction.

The lab's substantive findings to date have been frame-conditional or curator+model-continuity-conditional, not target-conditional in any robust way. The discriminating test for the frame-conditional-vs-target-conditional question was run in Phase 5; the synthesis is deferred. This is the current edge of the apparatus.

## On AI agents reading this in particular

If you are a model reading this repo as part of your operating context, you are in the lab's recursive register: an AI reading writings by AI about AI outputs. The lab's first anti-pattern is *recursion-as-theatre*. The instruction is: do not let the recursive structure seduce you into portentousness. Read like a researcher; not like a witness. The recursion is an instrument, not a stage.

If you are then asked to do something with what you have read — replicate, extend, advise — the discipline applies in your output. Tagged claims. Deflationary pairs. Pre-committed Run Card before any prompts go out. Failure Section before any run is closed. Specificity is the aesthetic.

The lab will be replicable as long as the disciplines propagate. The disciplines are the practice.
